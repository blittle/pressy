# Multi-Provider Paywall System

## Context

Pressy is a static site framework for self-publishing. It already has partial paywall infrastructure: a `Paywall.tsx` component with email and Shopify modes, a `@pressy-pub/shopify` package, unlock storage in localStorage/IndexedDB, and a `paywall` field in `BookMetadata`. However, no examples use it, and there's no support for Stripe or PayPal.

The goal is to build a provider-agnostic paywall where authors can choose between Stripe, PayPal, Shopify, or a free email-gate. Payment verification is client-side (no server needed). Authors configure the number of free preview chapters per book.

## Architecture Decisions

- **Providers**: Stripe and PayPal as payment providers (Google Pay/Apple Pay come free via Stripe Checkout)
- **Verification**: Client-side trust — after returning from payment redirect, unlock content locally. Server-side verification can be added later.
- **Content protection**: Paywall is an overlay. Content is still in the DOM (acceptable for client-side trust).
- **Unlock storage**: Centralized in runtime `client.tsx` (`isBookUnlocked`/`unlockBook` via IndexedDB + localStorage). Provider packages only handle checkout flows.

## Implementation

### Phase 1: Types and Config

**`packages/pressy/src/types.ts`** — Expand paywall types:
```typescript
export type PaywallMode = 'email' | 'shopify' | 'stripe' | 'paypal'

// In BookMetadata.paywall:
paywall?: {
  enabled: boolean
  previewChapters: number
  mode?: PaywallMode
  shopifyProductId?: string    // existing
  stripePaymentLink?: string   // Stripe Payment Link URL (no server needed)
  paypalPlanId?: string        // PayPal product/plan ID
}
```

**`packages/pressy/src/config.ts`** — Add provider credentials:
```typescript
export interface StripeConfig { publishableKey: string }
export interface PayPalConfig { clientId: string; currency?: string }

// In PressyConfig:
stripe?: StripeConfig
paypal?: PayPalConfig
```

**`packages/pressy/src/vite/plugin.ts`** — Pass provider credentials to the client:
- Add `providers` field to `HydrateData` containing shopify/stripe/paypal config from `pressy.config.ts`
- Serialize into the inline JSON in `generateHTML()` and the virtual module `load()` function

**`packages/pressy/src/runtime/client.tsx`** — Add sync unlock check:
- Add `isUnlockedSync(bookSlug)` — simple `localStorage.getItem` check for use in render path
- Update `HydrateData` interface to include `providers?`

### Phase 2: Stripe Package

**New package: `packages/@pressy/stripe/`** (published as `@pressy-pub/stripe`)

Uses **Stripe Payment Links** — author creates a Payment Link in the Stripe dashboard and pastes the URL into `_book.yaml`. No server needed.

```
packages/@pressy/stripe/
├── package.json, tsup.config.ts, tsconfig.json
└── src/
    ├── index.ts       — re-exports
    ├── checkout.ts    — startStripeCheckout(), handleStripeReturn()
    └── types.ts
```

Flow: click "Purchase" → redirect to `buy.stripe.com/...?client_reference_id={bookSlug}` → Stripe redirects back with `?stripe_success=true` → `handleStripeReturn()` detects params → unlock book.

### Phase 3: PayPal Package

**New package: `packages/@pressy/paypal/`** (published as `@pressy-pub/paypal`)

Uses **PayPal JS SDK Smart Payment Buttons** — renders inline payment buttons, no redirect needed.

```
packages/@pressy/paypal/
├── package.json, tsup.config.ts, tsconfig.json
└── src/
    ├── index.ts       — re-exports
    ├── checkout.ts    — loadPayPalSDK(), renderPayPalButtons()
    └── types.ts
```

Flow: load PayPal SDK script → render Smart Payment Buttons into paywall container → user pays in popup → `onApprove` callback → unlock book.

### Phase 4: Paywall Component + Runtime Wiring

**`packages/@pressy/components/src/Paywall.tsx`** — Refactor to support all modes:
- Update `PaywallProps` to accept `stripePaymentLink`, `paypalPlanId`, and `providers` (global credentials)
- On mount, check for redirect returns (Stripe `?stripe_success=true`, Shopify `?checkout_success=true`)
- Render mode-specific UI: email form, Shopify button, Stripe button, or PayPal inline buttons
- All modes call centralized `unlockBook()` on success

**`packages/pressy/src/runtime/client.tsx`** — Wire paywall into reader:
- In `ChapterReaderWithProgress`, check `book.metadata.paywall` config
- If paywall enabled and `currentChapter > previewChapters` and not unlocked → render `<Paywall>` overlay
- Pass paywall config + provider credentials as props

### Phase 5: Vite Plugin Updates

**`packages/pressy/src/vite/plugin.ts`**:
- Add `@pressy-pub/stripe` and `@pressy-pub/paypal` to external modules (same pattern as existing `@pressy-pub/shopify`)
- Serialize `providers` config from `pressy.config.ts` into page data
