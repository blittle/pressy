# Bookmarking Feature

## Context

The reader currently only persists the user's last-read position. We want to add bookmarking so users can save multiple positions and jump back to them. The UX: a bookmark ribbon icon in the footer bar that opens an expandable panel (matching the existing settings panel pattern) showing saved bookmarks + a "Bookmark this page" button.

## Files to Modify

1. `packages/pressy/src/types.ts` — Add `Bookmark` interface
2. `packages/@pressy/components/src/reader/types.ts` — Add `BookmarkData` and `BookmarkProps` types
3. `packages/pressy/src/runtime/client.tsx` — DB_VERSION bump, IndexedDB helpers, bookmark callbacks, wire props to Reader
4. `packages/@pressy/components/src/Reader.tsx` — Pass `bookmarkProps` through to both readers
5. `packages/@pressy/components/src/index.ts` — Re-export new types
6. `packages/@pressy/components/src/reader/ScrollReader.tsx` — `bookmarksOpen` state, toggle button, panel UI
7. `packages/@pressy/components/src/reader/PaginatedReader.tsx` — Same as ScrollReader, adapted for paginated context
8. `packages/@pressy/components/src/reader/scroll-reader-styles.ts` — Bookmark panel CSS
9. `packages/@pressy/components/src/reader/paginated-reader-styles.ts` — Bookmark panel CSS

```ts
  bookSlug: string
  chapterSlug: string
  chapterTitle: string
  page: number
  totalPages: number
  scrollPosition: number
  createdAt: number
}

export interface BookmarkProps {
  bookmarks: BookmarkData[]
  onAddBookmark: (data: {
    chapterSlug: string
    chapterTitle: string
    page: number
    totalPages: number

  }) => void

}
```

## Step 2: IndexedDB + callbacks in `client.tsx`

**2a.** Bump `DB_VERSION` from `1` to `2`. Add `const BOOKMARKS_STORE = 'bookmarks'`.

**2b.** In `initDB`'s `onupgradeneeded`, add the new store with the existing `if (!contains)` guard pattern:
```ts
if (!database.objectStoreNames.contains(BOOKMARKS_STORE)) {
  database.createObjectStore(BOOKMARKS_STORE, { keyPath: 'id' })
}
```

**2c.** Add helpers after `unlockBook()` (~line 112) following the exact same IDB transaction pattern as `saveReadingProgress`/`getAllReadingProgress`:
- `saveBookmark(bookmark: Bookmark): Promise<void>` — `store.put(bookmark)`
- `deleteBookmark(id: string): Promise<void>` — `store.delete(id)`
- `getBookmarksByBook(bookSlug: string): Promise<Bookmark[]>` — `store.getAll()` then filter by bookSlug, sort by createdAt desc

**2d.** In `ChapterReaderWithProgress`, add:
- `const [bookmarks, setBookmarks] = useState<Bookmark[]>([])`
- `useEffect` to load bookmarks on mount via `getBookmarksByBook(book.slug)`
- `handleAddBookmark` — creates Bookmark with id `${book.slug}:${data.chapterSlug}:${Date.now()}`, saves, refreshes state
- `handleDeleteBookmark` — deletes by id, refreshes state
- `handleNavigateBookmark` — same-chapter: scroll/page jump directly; cross-chapter: store restore data in `localStorage('pressy-bookmark-restore')` then `window.location.href`

**2e.** Pass `bookmarkProps` to `<Reader>`:
```tsx
bookmarkProps={{ bookmarks, onAddBookmark: handleAddBookmark,
  onDeleteBookmark: handleDeleteBookmark, onNavigateBookmark: handleNavigateBookmark }}
```

**2f.** In `handleRestoreProgress`, check `pressy-bookmark-restore` from localStorage BEFORE the existing `pressy-last-read` check, and remove the key after reading.
